# 修改UI介面清單10：確保轉錄區預設顯示空間（不用拉動）

## 1. 目標
解決在簡報模式下，使用者必須手動捲動（拉拉桿）才能看到完整三行轉錄文字的問題。
目標是**預設情況下**（開啟網頁或進入簡報模式時），下方轉錄區就有足夠的高度顯示「操作列」、「最新即時語音」以及「至少三行歷史轉錄紀錄」。

## 2. 影響範圍
- **樣式檔**：`css/style.css`
- **頁面**：`teacher.html`（簡報模式）

## 3. 調整項目

### 3.1 大幅調整版面比例 (Layout Ratio)
目前的 `5:3` 比例在部分螢幕高度下仍然不足。將調整為 **`1:1` (等比例)**，強迫將畫面一半的高度分配給下方轉錄區。
簡報區會自動縮放適應上半部空間，不會被裁切。

- `.container.presentation-mode .presentation-panel`
  - 原：`flex: 5 1 0`
  - 改：**`flex: 1 1 0`**
- `.container.presentation-mode .presentation-subtitle-panel`
  - 原：`flex: 3 1 0`
  - 改：**`flex: 1 1 0`**

### 3.2 設定最小高度保障 (Minimum Height)
為了防止在視窗極矮的情況下轉錄區仍被壓縮，將加入絕對最小高度限制。

- `.container.presentation-mode .presentation-subtitle-panel`
  - 新增：**`min-height: 350px;`**
  - 說明：這能確保容納 工具列(~50px) + 最新語音(~60px) + 歷史列表(~240px)，即使犧牲簡報大小也要保證文字可讀。

### 3.3 縮減垂直間距 (Compact Spacing)
微調內部元件的留白，讓每一寸空間都用在顯示文字上。

- `.live-transcription-block`
  - `padding: 1em` → **`padding: 0.5em 1em;`**
- `.stt-toolbar`
  - `margin-bottom: 8px` → **`margin-bottom: 4px;`**
- `.live-transcription-latest`
  - `margin-bottom: 10px` → **`margin-bottom: 6px;`**

## 4. 驗收標準
1.  在一般筆電或桌機螢幕開啟簡報模式。
2.  **不需要**去拉動右側的捲軸。
3.  畫面下方能清楚看到：
    -   錄音/深色模式按鈕列
    -   最新的一句即時語音框
    -   下方的列表中至少顯示 **3 筆** 完整的對話紀錄。

## 5. 回滾方案
若使用者覺得簡報畫面變得太小難以看清，可將比例回調至 `flex: 4` (簡報) : `flex: 3` (轉錄)。
