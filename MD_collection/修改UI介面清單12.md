# 修改UI介面清單12 - 調整預設版面與字幕區

## 目標
調整預設版面配置，使其在不縮放瀏覽器的情況下，達到類似「90%縮放」的視覺效果（更寬的內容區域），並確保字幕區域清晰且至少能顯示 3 行以上的大字體內容。

## 需求分析
1. **版面比例**: 使用者希望看到類似 90% 縮放的寬闊感。
   - 簡報區域應佔據主要畫面高度。
   - 字幕區域位於下方，高度固定但足夠顯示多行。
2. **字幕區域**:
   - 至少顯示 3 行以上。
   - 字體要大且清晰。
   - 不跑版。
3. **寬度調整**: 模擬縮放效果，增加主要內容容器的寬度佔比。

## 實作步驟 (CSS)

### 1. 調整容器寬度 (模擬 90% 縮放寬闊感)
- 修改 `.theater-container`:
  - `width`: 從 `90%` 增加至 `96%`。
- 修改 `.container.presentation-mode .video-list`:
  - 維持 `10%` 但確保最小寬度不佔用過多空間。

### 2. 調整垂直佈局 (簡報 vs 字幕)
目前設定為 `flex: 1` (50/50)，將改為簡報區佔據剩餘空間，字幕區固定高度。
- 修改 `.container.presentation-mode .presentation-panel`:
  - `flex`: 改為 `1` (佔據剩餘空間)。
  - `height`: `auto` (或 `100%`)。
  - `overflow`: `hidden` (避免撐開)。
- 修改 `.container.presentation-mode .presentation-subtitle-panel`:
  - `flex`: 改為 `0 0 320px` (固定高度 320px，確保空間足夠)。
  - `min-height`: `320px`。
  - `height`: `320px`。
  - 移除 `overflow-y: auto` (交由內部元件處理)。

### 3. 優化字幕顯示
- 修改 `.live-transcription-display`:
  - `font-size`: 增加至 `1.3rem` (約 21px，清晰大字)。
  - `line-height`: `1.6`。
  - `height`: `calc(100% - 60px)` (扣除工具列高度，填滿下方面板)。
  - `max-height`: `none` (移除最大高度限制，由面板高度決定)。
  - `min-height`: `200px`。

### 4. 驗證
- 確認在 100% 縮放比例下，簡報區寬大。
- 確認下方字幕區高度固定，且預設的 3 筆測試資料能完整顯示，不需要捲動（或剛好填滿）。
- 字體清晰易讀。
